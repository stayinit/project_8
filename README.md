# Описание REST API для сервиса аутентификации с помощью спецификации OpenAPI 3.0
## Исходные данные:
Задача заключалась в проектировании и документировании REST API для сервиса аутентификации и авторизации. Необходимо было охватить полный цикл работы с пользователем: от регистрации по номеру телефона до управления сессиями и профилем. Требовалось создать OpenAPI-спецификацию, включающую операции регистрации, подтверждения SMS-кода, входа/выхода из системы, обновления токенов, управления сессиями и получения профиля пользователя.
## Стек:
OpenAPI, REST API, Swagger, HTTP/HTTPS, JSON
## Методы и подходы:
- Анализ предметной области аутентификации и авторизации
- Проектирование REST API
- Использование стандарта OpenAPI 3.0 для документации
- Реализация механизмов безопасности через JWT токены
- Моделирование запросов и ответов в формате JSON
- Применение паттерна refresh token для обновления сессий
## Реализация проекта:
1. Разработана полная спецификация API из 8 endpoints:
- Регистрация: POST /auth/register/start, POST /auth/register/verify
- Авторизация: POST /auth/login, POST /auth/logout
- Управление сессиями: POST /auth/refresh, GET /auth/sessions, DELETE /auth/sessions/{session_id}
- Профиль: GET /auth/profile
2. Реализована двухэтапная регистрация:
- Начало регистрации с отправкой SMS-кода
- Подтверждение кода для завершения регистрации
3. Внедрена система безопасности:
- JWT токены для аутентификации (bearerAuth)
- Refresh token механизм для обновления сессий
- Защита чувствительных маршрутов через security schemes
4. Спроектированы структуры данных:
- Четко определенные схемы запросов для всех операций
- Детальные схемы ответов с примерами данных
- Обработка ошибок через соответствующие HTTP статусы
